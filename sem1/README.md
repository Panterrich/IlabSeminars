# 1. WSL, консоль

## Семинар

Linux - операционная система, стандарт де-факто во многих областях разработки. Является очень гибкой системой и предоставляет по-настоящему мощные и удобные инструменты для работы.

### WSL

Если у вас стоит операционная система Windows, то не нужно спешить ставить Linux вместо Windows или ставить вторую систему рядом. В Windows уже достаточно давно существует Windows Subsystem for Linux (WSL), которая в последние годы всё более активно развивается.

WSL - это функциональность Windows, которая позволяет использовать окружение Linux без виртуальной машины или наличия второй системы рядом (dual-boot).

#### Установка

Откройте Windows PowerShell от имени администратора. Для этого в поиске Windows наберите PowerShell и выберите "Запуск от имени администратора". Выберите "Да" в диалоге "Разрешить этому приложению вносить изменения на вашем устройстве".

![Запуск Windows PowerShell](images/powershell.png)

В окне PowerShell введите команду `wsl --install`. Установка начнётся, через какое то время возникнет ещё один диалог, в котором нужно опять ответить "Да".

После завершения установки потребуется перезагрузить компьютер (при наличии опции "Обновить и перезагрузить" нужно выбрать её).

Далее после перезагрузки запустите появившееся приложение "Ubuntu". Установка продолжится. В дальнейшем в этом приложении будет открываться терминал Linux для работы с WSL. Через некоторое время понадобится придумать и ввести имя пользователя, пароль и повторно ввести пароль (при вводе пароля символы не отображаются, в Linux это стандартная практика).

![Первый запуск приложения Ubuntu](images/wsl_installation.png)

На этом установка завершена. [Справка об установке на официальном сайте Microsoft](https://learn.microsoft.com/en-us/windows/wsl/install)

### Подключение к WSL с помощью VS Code

Работать с WSL можно прямо из VS Code. Для подключения к WSL нужно нажать на синюю кнопку в левом нижнем углу и в появившемся меню выбрать `WSL`. При первом подключении VS Code установит необходимые пакеты.

![Подключения к WSL в VS Code](images/vscode_wsl_connection.png)

После подключения к WSL VS Code будет работать с файлами внутри файловой системы WSL, а также открывать терминал WSL вместо терминала Windows. Определить, что VS Code подключен к WSL можно по наличии надписи "WSL: Ubuntu" в левом нижнем углу.

![Вид окна VS Code после подключения к WSL](images/vscode_wsl.png)

> [!NOTE]
> WSL обладает своей файловой системой, отдельной от Windows. Для доступа к файлам Windows в директории `/mnt` находятся "примонтированные" директории дисков.
>
> Например, если файл в Windows расположен по пути `C:\Users\Alex\Desktop\hello.txt`, то в WSL его можно найти как `/mnt/c/Users/Alex/Desktop/hello.txt`. Также возможно получить доступ к файлам WSL из Windows. В проводнике появляется пункт "Linux".

> [!WARNING]
> Доступ к файлам Windows из WSL гораздо медленнее, чем доступ к файлам внутри самой WSL, поэтому рекомендуем нужные файлы скопировать в WSL.
>
> [Обсуждение данной проблемы на Github](https://github.com/microsoft/WSL/issues/4197)
